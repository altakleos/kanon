# Foundations

Cross-cutting source material this project commits to. Feature artifacts (specs, ADRs, plans) cite foundations rather than pass through them.

Three sub-namespaces:

- [`vision.md`](vision.md) — the product's identity, mission, non-goals.
- [`principles/`](principles/) — cross-cutting stances the project commits to, one per file.
- [`personas/`](personas/) — design-stressing user scenarios.

See [`../development-process.md`](../development-process.md) § Foundations for the method definition.
