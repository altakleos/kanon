# Vision — What `agent-sdd` Is, and What It Is Not

> **Status: bootstrap stub.** This document is authored fully in commit 2 of the bootstrap phase. Below is the one-paragraph north star so other boot-chain files can link here without breaking.

## What it is

`agent-sdd` packages a proven Spec-Driven Development (SDD) methodology as a pip-installable, self-hosting kit that works across any LLM-agent harness (Claude Code, Codex, Cursor, Copilot, Windsurf, Cline, Kiro, and more). A single pointer in the consumer repo's `CLAUDE.md` or `AGENTS.md` is enough for the agent to obey the kit's process gates — plan-before-build, spec-before-design, verification alongside implementation. The kit is tiered (tier-0 vibe → tier-3 platform) with non-destructive migration between tiers as projects grow.

## What it is not

It is not a project-management tool, a code generator, a formal-methods framework, or a CI linter. It is a prose-first, harness-portable SDD adoption kit whose artifacts are authoritative source (not documentation) in an AI-coded world where humans read specs more than code.
