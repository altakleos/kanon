# Plan: Kit bundle refactor + three protocols

## Context

Two changes land together (under `[Unreleased]`; no version bump):

1. **`src/kanon/templates/` → `src/kanon/kit/`** — restructure to mimic Sensei's `src/sensei/engine/` shape: one unified bundle with a kernel doc (`kit.md`), tunables YAML (`manifest.yaml`), and sub-buckets (`protocols/`, `files/`, `sections/`, `agents-md/`). Eliminates the ~4× duplication imposed by per-tier directories. See ADR-0011.
2. **Ship three prose-as-code protocols** scaffolded into consumer repos at `.kanon/protocols/`:
   - `tier-up-advisor.md` (tier-1+)
   - `verify-triage.md` (tier-1+)
   - `spec-review.md` (tier-2+)

   See ADR-0010 and `docs/specs/protocols.md`.

The two changes bundle because adding protocols to the old per-tier layout would grow the duplication problem, not solve it. The full design memo (context, alternatives, agent debate, resolved scope) lives in the session plan at `~/.claude/plans/implement-1-thru-7-whimsical-backus.md`; this file is the in-repo execution record.
