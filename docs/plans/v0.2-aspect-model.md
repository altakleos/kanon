# Plan: v0.2 — Aspect model refactor (SDD becomes the first aspect)

## Context

Promote aspects to first-class citizens in the kit per ADR-0012 + ADR-0013, with `sdd` as the single shipped aspect at this milestone. Zero user-visible behaviour change for existing v0.1 consumers: `kanon init foo --tier 2` continues to produce an identical tier-2 project; `kanon tier set` continues to work as sugar; legacy `.kanon/config.yaml` auto-migrates to schema v2 on first `upgrade`.

A follow-up plan introduces the first non-SDD aspect (`worktrees`) once this refactor lands — per the `solo-with-agents` persona, worktrees is the first aspect kanon's agent-first user pulls in.

See `docs/design/aspect-model.md` for the architecture this plan executes against.

- **`versioning` aspect** — second non-SDD aspect. Separate plan.
